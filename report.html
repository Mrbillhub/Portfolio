Personal Portfolio Page Development

Author: Rakesh Kumar Kushwaha

Date: 2025-05-19

Overview

This report outlines how I developed a personal portfolio website using only HTML5, based on the HTML Roadmap discussed in Week 2's lecture slides. The objective was to create a clean, accessible, and well-structured webpage that introduces me, lists my projects, and provides ways to contact me — all without using any CSS or JavaScript.

Structure

I organized the website using semantic HTML5 elements to improve readability and accessibility:

  • <header>: Contains my full name and a short professional tagline.
  • <main>: Divided into three key sections:
    • <section id="about">: A brief biography and a hyperlink to a more detailed "About Me" page.
    • <section id="projects">: Lists three placeholder projects with descriptions and a link to the Projects page.
    • <section id="contact">: Includes my email, social links, a hyperlink to the Contact page, and a simple contact form.
  • <footer>: Contains a simple copyright notice.

I also created separate pages for:

  • about.html My background, interests, and goals.
  • projects.html Descriptions of my current learning projects.
  • contact.html Contact form, email, social links, and a Google Map of my location.

Features

  • Used only HTML5 semantic tags like <header>, <main>, <section>, <article>, and <footer>.
  • Internal navigation using hyperlinks between pages.
  • Email and social links using mailto: and <a> tags.
  • Logical structure without CSS.
  • Embedded Google Map for location.

Learning

This project helped me understand how powerful semantic HTML can be, even without CSS. I learned how to:

  • Use structural tags properly for clarity and accessibility.
  • Logically divide content using <section> and <article>.
  • Link multiple HTML pages together effectively.
  • Create and structure a basic form using only HTML.

It was a great hands-on experience that built my foundation in web development. By focusing only on HTML, I improved my understanding of semantic structure and content hierarchy.
